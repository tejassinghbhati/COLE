# Cost of Living Explainer

An ML-powered tool for **understanding** what drives the cost of living across cities —
not just predicting a number, but explaining *why* a place is expensive and *who* can
actually afford to live there.

> Part of a series of apps that help people understand socioeconomic problems using
> machine learning.

> 📖 **Want the science?** [`docs/HOW_IT_WORKS.md`](docs/HOW_IT_WORKS.md) is a full,
> diagram-rich walk-through: the data source, how indices are re-based, how XGBoost is
> trained, the SHAP math, and every formula behind the numbers on screen.

## What it does

- **Real data.** 545 cities from **Numbeo's** cost-of-living indices, re-based so that
  **New Delhi = 100** — every city is described relative to Delhi.
- **Predicts** a city's total cost of living from its underlying indices (rent, groceries,
  restaurants) and local purchasing power.
- **Explains** each prediction with **SHAP** values — exactly which factors push a city
  above or below the typical city. This is the core "understanding" feature.
- **Per-city overview.** A short, plain-language summary written for *each* city —
  generated by an **LLM (Claude)** when an API key is present, with a data-driven template
  fallback so it always works offline.
- **Local purchasing power**: surfaces whether local wages keep up with local prices.
- **Finds similar cities** by cost *structure*, and **compares** two cities category by
  category.
- **Global insights**: which factors drive cost the most, across all 545 cities.

## Why this design teaches something

A model that just predicts a price forecasts — it doesn't explain. This app is built around
**explainable ML**: every prediction comes with a SHAP driver breakdown, and every city gets
a personalized written overview, so users learn *what* makes cost of living high and *how*
wages change the real burden.

## Quick start

First clone the repo and move into the backend:

```bash
git clone https://github.com/tejassinghbhati/COLE.git
cd COLE/backend
```

**Windows / PowerShell**

```powershell
python -m venv .venv
.\.venv\Scripts\Activate.ps1
pip install -r requirements.txt

python -m scripts.build_dataset        # 1. build real dataset (New Delhi = 100)
python -m scripts.generate_overviews   # 2. write per-city overviews
python -m scripts.train                # 3. train model + SHAP explainer
uvicorn app.main:app --reload          # 4. run
```

**macOS / Linux**

```bash
python3 -m venv .venv
source .venv/bin/activate
pip install -r requirements.txt

python -m scripts.build_dataset        # 1. build real dataset (New Delhi = 100)
python -m scripts.generate_overviews   # 2. write per-city overviews
python -m scripts.train                # 3. train model + SHAP explainer
uvicorn app.main:app --reload          # 4. run
```

Then open <http://127.0.0.1:8000>

## Enabling LLM-written overviews

Overviews use a deterministic, data-driven template by default (no key needed). To get
**Claude-written** overviews instead, set your API key and re-run the overview step:

```powershell
# Windows / PowerShell
$env:ANTHROPIC_API_KEY = "sk-ant-..."
python -m scripts.generate_overviews   # re-generates via Claude Haiku, caches to JSON
```

```bash
# macOS / Linux
export ANTHROPIC_API_KEY="sk-ant-..."
python -m scripts.generate_overviews   # re-generates via Claude Haiku, caches to JSON
```

No other change is required — the API serves whatever is in `data/overviews.json`.

## Data & provenance

Source: **Numbeo — Cost of Living Index by City** (current rankings), the most widely cited
cross-city reference. Numbeo publishes indices relative to New York City = 100;
`build_dataset.py` re-bases every index to **New Delhi = 100**. A snapshot of the source is
cached in `data/numbeo_raw.html` for reproducibility. To refresh, delete that file and
re-run `build_dataset.py` (it will re-fetch live).

## Model at a glance

| | |
|---|---|
| **Task** | Regress total cost of living (New Delhi = 100) from rent / groceries / restaurants / purchasing power |
| **Model** | XGBoost gradient-boosted trees |
| **Explainability** | SHAP (TreeExplainer) — exact per-prediction attribution |
| **Similarity** | Standardized k-nearest-neighbours on cost structure |
| **Hold-out** | R² ≈ 0.99, MAE ≈ 6.5 index points (545 cities, 116 countries) |

FastAPI · XGBoost · SHAP · scikit-learn · pandas · Anthropic (optional) — vanilla JS frontend.
